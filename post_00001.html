pythonで動く静的サイトジェネレータ "pelican"

自分で1からサイト作れるようになりたいなあとずっと思っていたのでついにサイト制作に手を出しちゃいました。サイト制作にあたって、pythonが好き（というかpythonしか環境構築してないから選択肢がない）なのでpelicanというpythonで動く静的サイトジェネレータを使ってみることにしました。

ところがこのpelicanというライブラリ、なかなかまとまった日本語の情報が落ちていません。

じゃあもう自分の備忘録用のブログを作るついでに、まとまったpelicanの解説記事群を作ってしまおうと思い立ちました。

そこで、今回はpelicanでサイトを作成し、github pagesを使って実際に公開するまでの実際の手順をご紹介します。というよりは、自分でやってみたことの記録です。

github pagesを使う理由はgitをちゃんと触ってみたかったのと、サーバー代もドメイン代もかからないのがアドすぎるからです。

githubのリポジトリはpublicならいくらでも作れる（はず）ので、ソースコードを公開すればサイト自体は規約の範囲内で無料で作り放題（な気がする）！

GitHub Pagesについて

一応pelicanを触り始めた時の筆者のプログラミングスペックも記載しておきます。

Python: github copilot（AIアシスタント）を使ってかろうじて書ける
html: 読めばだいたい何言ってるかはわかるが自分で書くのはしんどい
css: 全く知らないし、完全に理解していない
javascript: 本当に何も知らないし、型付けがめんどそう（Pythoner並感）

あ、あと環境としてはwindowsでの話になります。
MACだとコマンドとかの部分がちょっと変わるのかも。

静的サイトと動的サイト

この世のサイトは2つしかない。静的サイトかそれ以外かだ。

このインターネット上のウェブサイトは大きく「静的サイト」と「動的サイト」に分けられます。

あんまりちゃんと理解はしていませんが、静的サイトはウェブサイトの基本セット（html + css + javascript）で構成されている、シンプルで誰がいつ見ても変わらないやつのことで、動的サイトはPHPとかを使ってサーバーといっぱいデータ通信をするやつって捉えています。

速い遅いとかまだそんなに分かっていないのですが、まあ原理的に静的サイトの方が軽くて当然ですよね。有名なものとして阿部寛のホームページがよくネタとして擦られています。

個人ブログの選択肢としてよくWordPressが挙げられますが、あれはPHPで動いてるみたいです。無料テーマはちょっとダサい感じするし、カスタマイズ性もしんどいです。そもそも個人ブログでサーバーサイドとデータのやり取りを沢山するケースがよくわからないので、静的サイトで作れるならそっちの方がよいと思っています。

問題は、静的サイトの基本セット（html + css + javascript）を何もないところから自分で組み立てたり、ブログとして新しい記事を安定して投稿できるようにするのはちょっとガチでしんどいということです。copilotとかChatGPT使っても流石に限界がある。

静的サイトジェネレーター、それは魔法のアイテム

ウェブサイト作りたいなーと思って色々調べている内に、markdown記法で書いたものをhtmlに変換する技術や、予め用意したcssのテンプレートなどを使ってサイトを制作できるプログラムがあることを知りました。

やってることは原理的にはnoteとかqiitaみたいな文章コンテンツ投稿サイトのフロントエンドと同じなのかなと思ってます。（間違ってたらごめん。でもqiitaなんてそもそも記事をmarkdownで書くんだし。）

ちまちまhtmlタグを一行一行打ち込むのはブログとして現実的に不可能ですし、markdown記法で書いたものを自分でも簡単に変換できるのは革命だと思いました。

なんなら一度サイトの基本形を作ってしまえばあとはmarkdownで書いた記事を更新するだけなので、やることが単純明快でいいなと感じました。WordPressはごちゃごちゃしたUIにPHPとかいう謎言語の時点でもうやる気がなくなりますよね。

有名な静的サイトジェネレータとしてはHugoやJekyllといったものがあるらしいのですが、Pythonで動くということと、pelicanという間抜けな名前に引かれて（そして日本語の情報があまりないマイナーな感じが面白くて）こいつを選びました。

サイトを公開するまでの簡易手順 【pelican編】

サイトを作って公開するための手順についてなのですが、実際はコンテンツやデザインを作り込む作業があるのを省略して、単純にpelicanでmarkdownからサイトを生成してgithubにぶち込む手順の紹介になります。

まずはpelicanがどういうものなのか、github pagesってどう使うのかといったことの確認です。ぶっちゃけこれぐらいの内容はqiitaとかにもあるのですが、一応最初から丁寧に記録していきます。

必要なライブラリをインストール

まずは仮想環境をvenvで作成しましょう。
この程度の処理に仮想環境が必要なのかはわかりませんが、万が一のため。
適当なフォルダを用意し、そこでコマンドプロンプトから以下を実行。

python -m venv venv

これでvenvという仮想環境のフォルダが作成されるので、そのフォルダと同じディレクトリにprojectフォルダを作ってしまいましょう。手動で作ってもコマンドプロンプトから作っても構いません。

md project

以下からの手順は全て、projectフォルダをカレントディレクトリとして話を進めます。

仮想環境をアクティベートします。
パスを指定してactivate.batを実行。

..\venv\Scripts\activate.bat

(venv)と表示されて入れば仮想環境のアクティベートに成功です。

ここに必要なライブラリをpipコマンドでインストールします。

pip install pelican markdown ghp-import

これで仮想環境への必要なライブラリのインストールが完了しました。
前の2つはpelicanでmarkdownkからサイト自体を作るためのもの、ghp-importはgit関係のものです。

markdownで記事を書いてpelicanでサイト化

次に、以下のコマンドを実行します。

pelican-quickstart

これをやるとなんか設定用にめっちゃ質問が飛んでくるので以下のように答えていきます。（github pagesで公開するための設定例です。）

What will be the default language of this web site? [English] ja

Do you want to enable article pagination? (Y/n) Y
How many articles per page do you want? [10] 10
What is your time zone? [Europe/Paris] Asia/Tokyo

Do you want to upload your website using FTP? (y/N) N

Do you want to upload your website using Dropbox? (y/N) N
